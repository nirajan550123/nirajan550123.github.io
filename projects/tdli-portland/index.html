GIS Case Study

Transit Dependency Likelihood Index – Portland, Oregon

GIS-based Transit Dependency Likelihood Index (TDLI) and service gap analysis built to support equity-focused transit planning.

ArcGIS Pro ArcPy Python Pandas Network Analyst

Project overview

This project develops a Transit Dependency Likelihood Index (TDLI) for Portland, Oregon and uses transit service areas to identify high-need neighborhoods that are poorly served by bus stops.

The workflow combines demographic indicators, z-score standardization, weighted scoring, and network-based service areas to help planners see where transit investment could reduce inequities in access.

The main goal was to create a reusable GIS workflow that links demographic need with actual transit coverage and highlights service gaps in a way that is easy to map and interpret.

Role & Scope

What I did

  • Designed the TDLI concept and selected demographic variables.
  • Implemented the workflow in ArcGIS Pro and ArcPy.
  • Built a reusable script-based analysis process for future cities.
  • Produced maps of high-TDLI areas and unserved street segments.

Problem & goal

Many cities have relatively strong transit systems overall but still leave some neighborhoods underserved, especially communities with higher dependency on public transportation.

The project goal was to build a GIS tool that could:

  • Quantify transit dependency at the neighborhood level.
  • Map where high-dependency areas lack nearby bus stops.
  • Output unserved street segments for targeted improvements.

Data & study area

The study area is the City of Portland, Oregon. The analysis uses:

  • Census-based demographic variables such as age, disability, poverty, and vehicle access.
  • Portland city boundary and census geography.
  • Bus routes and stop locations.
  • Street network and a network dataset for service areas.

These variables are transformed into indicators of transit dependency and combined into a single TDLI score for each polygon.

Methodology

  1. Export selected fields from the polygon layer into a pandas DataFrame using ArcPy data access.
  2. Standardize each variable using z-scores and reclassify values into W-scores of 0, 0.5, and 1.
  3. Compute a weighted TDLI score for each polygon using equal or custom weights.
  4. Generate service areas around bus stops using Network Analyst and a walking-distance cutoff.
  5. Select polygons with TDLI ≥ 0.5 and identify the portions outside the transit service area.
  6. Clip the street network to those unserved polygons to output unserved street segments.

Key outputs

  • All polygons with TDLI values and supporting fields.
  • High-TDLI polygons where TDLI ≥ 0.5.
  • Unserved high-TDLI polygons outside service buffers.
  • Unserved street segments in high-need areas.

Results & insights

The TDLI maps highlight neighborhoods with higher transit dependency, while the gap analysis shows which of those neighborhoods do not fall within the walking-distance service area of existing bus stops.

The unserved street layer provides a direct way to identify where additional stops, routing changes, or micro-transit options could improve equity in transit access.

What I would improve next

  • Use mode-specific accessibility thresholds for bus, light rail, and commuter rail.
  • Incorporate service frequency and reliability so high-frequency routes are weighted more heavily.
  • Add preprocessing checks for data quality, missing values, and coordinate system consistency.
  • Package the workflow as a Python toolbox (.pyt) for easier reuse in other cities.
