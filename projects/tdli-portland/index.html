Transit Dependency Likelihood Index – Portland, Oregon

GIS-based Transit Dependency Likelihood Index (TDLI) and service gap analysis for equity-focused transit planning.

Tools: ArcGIS Pro · ArcPy · Python · Pandas · Network Analyst

Project overview

This project develops a Transit Dependency Likelihood Index (TDLI) for Portland, Oregon and uses transit service areas to find high-need neighborhoods that are poorly served by bus stops.

The workflow combines demographic indicators, z-score standardization, weighted scoring, and network-based service areas. The outputs help planners see where transit investment could reduce inequities in access.

Role & scope

What I did

>Designed the TDLI concept and variable set >Implemented the workflow in ArcGIS Pro and ArcPy >Built a reusable script tool for future cities >Produced maps of high-TDLI and unserved streets

Problem & goal

Many cities have good transit overall but still leave some neighborhoods underserved, especially communities with higher dependency on public transport. The goal of this project was to build a GIS-based tool that:

    >Quantifies transit dependency at the neighborhood level >Maps where high-dependency areas lack nearby bus stops >Outputs unserved street segments for targeted improvements

Data & study area

The study area is the City of Portland, Oregon. The analysis uses:

    >Census-based demographic variables (age, disability, poverty, vehicle access, etc.) >Portland city boundary and census geography >Bus routes and stop locations >Street network and a network dataset for service areas

Demographic variables are transformed into indicators of transit dependency, then combined into a single TDLI score for each polygon.

Methodology

> Export selected fields from the polygon layer into a pandas DataFrame using ArcPy data access. > Standardize each variable using z-scores and reclassify to W-scores (0, 0.5, 1) based on thresholds. > Compute a weighted TDLI score for each polygon from the W-scores (equal or custom weights). > Use Network Analyst to generate service areas around bus stops using a walking-distance cutoff. > Select polygons with TDLI ≥ 0.5 and identify areas outside the service area (unserved high-TDLI polygons). > Clip the street network to these unserved polygons to output unserved street segments.

Key outputs

>All polygons with TDLI values and supporting fields >High-TDLI polygons (TDLI ≥ 0.5) >Unserved high-TDLI polygons outside service buffers >Unserved street segments in high-need areas

Results & insights

The TDLI maps highlight neighborhoods with higher transit dependency, while the gap analysis shows which of those neighborhoods do not fall within the walking-distance service area of existing bus stops.

The unserved street layer provides a direct way to see where additional stops, routing changes, or micro-transit options could improve equity in access to transit.

What I would improve next

    > Use mode-specific accessibility thresholds (e.g., different walking distances for bus, light rail, and commuter rail). > Incorporate service frequency and reliability so high-frequency routes are weighted more than infrequent service. > Add preprocessing checks for data quality, missing values, and consistent coordinate systems. > Package the workflow as a Python toolbox (de>.pyt) for easier reuse by planners in other cities.
